PawMind

AI 宠物健康管理平台 — 让 AI 成为宠物的私人健康管家

产品定位：PawMind 是面向城市养宠人群的 AI 宠物健康管理 App，通过智能设备数据采集、健康指标可视化、MoE 架构 AI 健康问答、拍照诊断和健康知识服务，帮助主人全面掌握宠物健康状况，及时发现健康隐患，低门槛获取可信的宠物健康建议。

核心数据流

多渠道健康数据（设备自动上报 / 手动录入 / 拍照扫描）
  → 数据聚合与指标计算
  → 健康指标可视化（24h / 7天趋势）
  → CareAI MoE 智能体（意图路由 → 专家分发 → RAG 检索 → LLM 生成 → 记忆管理）
  → 主人知情 & 行动（预警 / 可信回答 / 就医建议）

二、用户与使用场景

2.1 典型用户画像

用户类型特征描述核心需求
忙碌打工人 25–35 岁城市白领，独居养宠，工作日在公司超 10 小时 快速了解宠物状态，低门槛陪伴感
健康焦虑型主人 对宠物健康高度重视，愿意记录详细数据 全面追踪健康指标，异常预警，就医时能提供完整病史
新手宠主 刚开始养宠，不了解宠物行为规律与护理常识 饲养建议，健康知识，AI 辅助判断是否需要就医
多设备用户 已购置智能喂食器、项圈、猫砂盆等硬件 设备统一管理，数据自动同步，无需分散查看

2.2 核心用户旅程图

阶段触点用户行为痛点/情绪功能转化
1. 发现 社交媒体/口碑 看到"AI 宠物健康"相关内容 半信半疑，不确定价值 健康报告截图社交传播，营造社交货币
2. 注册建档 App 首次启动 填写宠物信息、上传照片 流程繁琐，担心无聊放弃 3 步建档，AI 生成宠物性格速写
3. 设备绑定 设备管理页 购置设备后绑定到 App 不确定兼容性，绑定步骤不直觉 设备型商城 + 图文配对引导
4. 日常监测 健康 Tab 每天查看健康指标卡 数据散乱，无法一眼判断宠物状态 健康指标卡（24h/7天汇总 + 异常标注）
5. 异常处理 推送通知 收到异常提醒，查看详情 不知严重程度，慌乱不知如何应对 AI 分级预警 + 分级行动建议
6. AI 健康咨询 CareAI Tab 发现异常或日常疑问时咨询 AI 网上信息杂乱，无法判断可信度；AI 回答不准确 CareAI MoE 问答：专家路由 + RAG 知识检索 + 记忆连续对话
7. 成长记录 成长册 记录精彩瞬间，分享给朋友 照片散乱，无成就感和仪式感 AI 月度成长报告，一键分享

三、核心页面 v2.1 新增

页面 1：首页（今日状态）

用户打开 App 的第一视角，快速掌握宠物今日整体状态与最新提醒。

区域内容设计要点
顶部导航宠物昵称/切换、通知铃、头像多宠物用户可快速切换，通知铃显示未读气泡
今日状态卡AI 生成的宠物今日健康一句话摘要 + 健康评分情感化表达，让主人一眼感知宠物状态
快捷指标3~4 个核心指标横向滑动展示点击跳转健康 Tab 对应指标详情
最新提醒异常预警 / 疫苗到期提醒 / 每日小知识按优先级排序，预警红色高亮
底部 TabBar首页 / CareAI / 健康 / 成长册 / 我CareAI 居中强调，凸显核心差异化功能

页面 2：CareAI（AI 健康助手）

宠物健康问答与 AI 诊断的核心入口，v2.1 升级为 MoE 架构后的产品呈现页。

区域内容设计要点
每日小知识顶部横幅展示今日知识卡片按专家分类着色，与 MoE 专家域对应
AI 诊断入口宫格口腔 / 粪便 / 皮肤 / 报告解读 / 药品识别图标 + 简短描述，一目了然
悬浮问答按钮右下角"向 CareAI 提问"快捷入口保持随时可达性；问答页面为全屏对话 UI
专家标识AI 回复气泡左侧显示路由到的专家图标增强可信度感知，让用户知道回答来自哪个专家

页面 3：健康（指标中心）

宠物健康数据的全量视图，支持设备数据与手动录入双轨并行。

区域内容设计要点
健康指标卡横列饮食/饮水/运动/睡眠/体温/心率/排便/体重横向滑动；已激活显示数值，未激活显示引导购买
趋势折线图点击指标卡展开 24h/7天趋势图异常区间红色标注，正常区间绿色底纹
手动录入入口右上角"+ 录入"按钮支持体重/饮食量/饮水量/体温/排便快速录入
异常预警卡当前活跃预警列表点击可查看 AI 建议 + 行动选项

页面 4：成长册

宠物成长历程的情感化记录，承载照片、视频与里程碑。

区域内容设计要点
月度成长报告AI 生成的图文月报（体重/活动量/健康摘要）分享卡设计，适合社交媒体发布
时间线按月分组展示照片/视频/里程碑里程碑特殊视觉标记（如：第一次洗澡）
上传入口"+ 添加"浮动按钮，可拍照或从相册选择快速录入，不打断用户情绪

页面 5：我（个人中心）

用户账号管理、宠物档案设置、设备管理、疫苗本的统一入口。

区域内容设计要点
宠物档案当前宠物基础信息查看/编辑头像大图展示，信息完整度引导补充
设备管理已绑定设备列表 + 添加设备入口设备状态一眼可见（在线/离线/低电量）
电子疫苗本疫苗接种历史 + 下次提醒日期待接种疫苗置顶红色提醒
健康档案就诊记录 + 日常观察时间线支持多输入方式（打字/OCR/语音）

5.1 健康管理 — 数据可视化

flowchart TD A([用户进入健康 Tab]) --> B[加载宠物绑定设备列表] B --> C[查询 24h/7天健康指标数据] C --> D[渲染健康指标卡区域] D --> E{指标是否已激活?} E -- 已激活 --> F[展示数值 + 趋势图] E -- 未激活 --> G[展示营销引导卡] G --> H{用户点击?} H -- 是 --> I[跳转设备商城] F --> J{用户点击指标卡?} J -- 是 --> K[展开指标详情 + 折线图] K --> L{切换时间范围?} L -- 24h --> M[展示 24 小时数据] L -- 7天 --> N[展示 7 天数据]
规则说明：
  • 已激活指标显示：指标名称、当前值、单位、对比上一周期变化（↑/↓/持平）
  • 异常区间在折线图中红色标注，正常区间绿色底纹
  • 手动可录指标在详情页提供「手动录入」快捷入口

8.2 完整请求管道（⑧ 步）

①
加载记忆
Redis 取最近 6 轮短期对话（按 sessionId）；PostgreSQL 取该宠物长期记忆摘要（按 petId）
②
Query Rewriting + 意图路由（1 次 LLM 调用）
输入：原始问题 + 短期历史 + 宠物档案摘要；输出：{ expert, confidence, rewritten_query }；confidence < 0.75 → 路由兜底专家
③
向量检索
ChromaDB 按 { expert, species } 预过滤，检索 top-10 相关知识片段
④
BGE Rerank
Cross-Encoder 精排（BAAI/bge-reranker-base，本地运行），保留 top-3 知识片段
⑤
检索置信度门控
最高相似度 ≥ 0.75 → 正常生成；0.5~0.75 → 降级生成（附不确定性声明）；< 0.5 → 直接返回兜底话术，跳过 LLM 调用
⑥
组装 Prompt
System Prompt（专家人设 + 硬约束）+ 长期记忆摘要 + 宠物档案 + RAG 知识片段（top-3）+ 短期对话历史（最近 6 轮）+ 改写后的问题
⑦
LLM 生成回答
国内模型 API（DeepSeek / 通义千问 / 智谱 GLM）；所有专家硬约束：不能编造医疗建议，严重症状必须附就医提示
⑧
异步更新长期记忆
判断本轮是否产生新事实（确诊/过敏/健康事件/跟进建议）→ 是则追加长期记忆摘要（1 次轻量 LLM 调用）；否则跳过

类型存储内容生命周期

8.4 幻觉控制

相似度区间策略是否调用 LLM
≥ 0.75 正常生成，基于知识片段回答 ✅ 是
0.5 ~ 0.75 降级生成，附不确定性声明：「以下仅供参考，建议咨询专业宠物医生确认」 ✅ 是
< 0.5 直接返回兜底话术，不生成（节省成本）：「这个问题超出了我目前的知识范围，建议带 [宠物名] 去宠物医院做专业检查」 ❌ 否
所有专家 System Prompt 硬约束：
  • 只能基于「参考知识」中提供的内容作答
  • 若参考知识不足，必须明确告知用户「我暂时没有关于这个问题的可靠信息」
  • 绝对不能凭空推测或编造医疗建议
  • 对于严重症状（出血、抽搐、呼吸困难、超过 24 小时不进食），必须在回答末尾附上「请立即就医」
  • 若问题与宠物健康无关，礼貌拒绝并引导回正题

维度指标目标值

差异化亮点

亮点一：MoE 专家路由 + RAG，可信的宠物健康知识问答

通用 AI 助手对宠物健康问题的幻觉率高、边界不清晰；现有宠物 App 的问答停留在关键词匹配层面。PawMind CareAI 的根本区别在于：用专领域知识库 + MoE 路由架构保证回答的可信度，用三档置信度门控防止输出不确定内容。
  • 5 个领域专家（症状/饮食/疫苗/行为/护理）+ 通用兜底，问题精准分发
  • 三档置信度门控：知识不足时主动承认，不编造医疗建议
  • 严重症状硬约束触发就医提示，产品级安全保障
  • RAGAS 框架持续评估 RAG 质量，不靠感觉迭代

亮点二：两层记忆管理，真正的连续对话体验

现有宠物 AI 每次对话都从头开始，主人需要反复说明宠物情况。PawMind CareAI 通过短期 + 长期两层记忆，让 AI 像真正认识这只宠物一样持续跟进健康状况。
  • 短期记忆（Redis）：保留当次 session 最近 6 轮，覆盖完整症状描述链
  • 长期记忆（PostgreSQL）：AI 自动提炼关键健康事实，跨 session 持久化
  • 新 session 自动载入宠物档案 + 长期记忆，对话立刻有上下文

亮点三：AI 健康管家，数据驱动决策

现有产品健康功能停留在数据记录层面，遇到异常主人仍然不知道怎么办。PawMind 不只记录数据，还告诉你该怎么做。
  • AI 分级预警直接给出行动建议（观察/调整饮食/立即就医）
  • 预警一键接入 CareAI 详细咨询，从发现到解决闭环
  • 健康档案历史数据可在就医时提供给宠医参考

亮点四：设备即服务，软硬一体化

健康指标卡与智能设备深度绑定，未绑定 = 营销机会，绑定 = 数据驱动价值闭环。
  • 已绑定设备：指标卡显示实时数据 + 趋势
  • 未绑定设备：指标卡显示「解锁功能」引导 + 设备商城直达
  • 一站式设备管理，无需多 App 切换

十、设计原则

原则说明
数据驱动决策 健康指标给出明确的正常/异常判断和行动建议，不让主人自己解读原始数据
AI 透明可信 所有 AI 分析结果附带免责声明；知识不足时主动承认，不过度承诺 AI 能力；置信度低时明确告知用户
设备即服务 未绑定设备 = 营销机会，用「解锁功能」的方式引导用户购买设备；不强制要求设备才能使用基础功能
输入零门槛 提供多种输入方式（打字/OCR/语音/扫码），每种记录方式追求「3 秒内完成」
降级而非崩溃 AI 服务不可用时保留基础功能；CareAI 微服务故障时降级为关键词匹配；不因 AI 不可用让 App 整体瘫痪
情感优先于功能 当功能和情感体验冲突时，优先保证情感连接感；通知文案用拟人化表达，减少冷冰冰的系统感

11.3 系统边界说明

PawMind 移动端
    ↕ HTTP/JSON（现有 API 不变，移动端无感知 AI 架构）
PawMind NestJS 后端
    ↕ HTTP 内部调用（新增转发逻辑）
CareAI 微服务（Python FastAPI）
    ├── 意图路由器（LLM 分类）
    ├── 专家模块池（5 个领域 + 通用兜底）
    ├── 记忆管理层（Redis 短期 + PostgreSQL 长期）
    ├── RAG 管道（ChromaDB + BGE + Rerank + 门控）
    └── LLM 调用层（国内模型 API）
ChromaDB（本地向量数据库）
Redis（短期记忆）
PostgreSQL（长期记忆，复用 PawMind 现有数据库）
职责边界：
  • 移动端：发送问题 + sessionId，无需感知 AI 架构细节
  • NestJS：JWT 鉴权、从 DB 组装宠物档案、转发 CareAI 微服务、持久化消息
  • CareAI 微服务：纯 AI 逻辑（路由、专家调度、RAG、LLM、记忆管理），无直接鉴权职责
